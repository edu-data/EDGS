## 🎯 프로젝트 개요

경인교육대학교 학생의 **입학 → 재학 → 교육실습 → 졸업 → 임용 → 현장** 6단계 전주기 교육데이터를 단일 플랫폼에서 통합 관리하고, **임용률·충원율 KPI를 실시간 대시보드**로 제공하는 교육데이터 거버넌스 시스템입니다.

### 핵심 목표

| 코드 | 목표 | 설명 |
|------|------|------|
| G1 | 데이터 통합 | 분산된 입학·학적·실습·임용 데이터를 단일 플랫폼으로 통합 |
| G2 | 실시간 가시화 | 임용률, 충원율 등 핵심 KPI 실시간 대시보드 제공 |
| G3 | 환류 자동화 | 현장 수요·임용 패턴 분석 → 교육과정 개정안 자동 연계 |
| G4 | 개인정보 보호 | 역할 기반 접근 통제(RBAC), 데이터 익명화 |
| G5 | 확장성 | GAIM Lab(P2), K-Teacher Academy(P6) 연동 고려 모듈형 설계 |

## 🖥️ 데모 스크린샷

### D-01 종합 현황 (Executive Summary)

KPI 카드 4개 · 임용합격률 5년 추이 · 충원율 게이지 · 지역 분포 · 환류 알림

### 6개 대시보드 구성

| 화면 | 대시보드명 | 주요 기능 | 접근 역할 |
|------|-----------|---------|---------|
| D-01 | 종합 현황 | KPI 요약, 추이 차트, 환류 알림 | 총장·처장·관리자 |
| D-02 | 임용 분석 상세 | 합격률·소요기간·지역·전형 분석 | 교학처·취업팀 |
| D-03 | 충원율 모니터링 | 학부·대학원·캠퍼스별 충원율 | 입학처 |
| D-04 | 학생 생애주기 | 전주기 타임라인, 위험 학생 알림 | 지도교수 |
| D-05 | 비임용 진로 | 진로 다각화율, 마이크로디그리 연계 | 취업지원팀 |
| D-06 | 환류 리포트 | KPI 이탈 탐지, 교육과정 개선 추적 | 교육과정위원회 |

## 🚀 빠른 시작

- 인터넷 연결 (Apache ECharts CDN 로드용)
- 모던 브라우저 (Chrome, Edge, Firefox 최신 버전)
- Node.js / 서버 **불필요** — 단독 실행 HTML

### 실행 방법

```bash

git clone https://github.com/edu-data/EDGS.git

# 브라우저에서 열기 (더블클릭 또는 드래그 앤 드롭)
```

또는 **`index.html` 파일을 웹 브라우저에서 직접 열면** 즉시 실행됩니다.

## 📊 KPI 정의 (K01~K08)

| KPI | 지표명 | 산출 공식 | 목표값 |
|-----|--------|---------|-------|
| K01 | 학부 충원율 | 실입학자 / 입학정원 × 100 | 98% 이상 |
| K02 | 대학원 충원율 | 실입학자 / 대학원정원 × 100 | 80% 이상 |
| K03 | 임용 응시율 | 응시자 / 졸업생 × 100 | 현황 파악 후 설정 |
| K04 | 임용 합격률 | 합격자 / 응시자 × 100 | 전국 평균 상회 |
| K05 | 평균 임용 소요 기간 | 합격연도 - 졸업연도 평균 | 2년 이내 |
| K06 | 졸업 후 5년 임용률 | 5년 이내 합격자 / 졸업생 × 100 | 추적 관리 |
| K07 | 비임용 진로 다각화율 | 비임용 진로자 / 졸업생 × 100 | 현황 파악 후 설정 |
| K08 | 실습 평가 우수 비율 | 실습 상위 30% 학생 비율 | 60% 이상 |

## 🔄 환류 트리거 (T-01~T-06)

KPI가 임계값을 이탈하면 자동으로 담당자 알림이 발생하고 D-06 환류 리포트에 표시됩니다.

| 트리거 | 조건 | 액션 | 담당 |
|--------|------|------|------|
| T-01 | 임용합격률 < 전국평균 - 5%p | 교육과정위원회 긴급 안건 등록 | 교학처장 |
| T-02 | 학부 충원율 < 85% | 원인 분석 리포트 자동 생성 | 입학처장 |
| T-03 | 대학원 충원율 < 80% | 모집 전략 검토 요청 | 교학처장 |
| T-04 | 평균 임용 소요 > 3년 | 임용 준비 지원 강화 검토 | 취업팀장 |
| T-05 | 실습 우수 비율 < 50% | 실습 운영 방식 검토 | 실습팀장 |
| T-06 | 데이터 미입력 > 30일 | 담당 부서 독촉 알림 | 거버넌스 담당 |

## 🛣️ 구축 로드맵

```
Phase 0  준비 (1~2개월)   ──  PIA 실시, 거버넌스 조직 구성, 데이터 현황 조사
Phase 1  MVP ✅ 완료        ──  핵심 대시보드, 주요 KPI 산출 엔진
Phase 2  확장 (6~9개월)  ──  6단계 전체 테이블, NEIS 연동, GAIM Lab API
Phase 3  자동화 (10~12개월) ── 환류 자동화, 동문 설문, 데이터 품질 감사
Phase 4  고도화 (13개월~)  ──  AI 임용률 예측, K-Teacher Academy 연동
```

## 🔧 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|---------|
| UI/UX | HTML5 + Vanilla CSS | 서버 불필요, 즉시 실행 |
| 차트 | [Apache ECharts 5.4](https://echarts.apache.org/) | 고성능, 한국어 지원 |
| 폰트 | Google Fonts — Noto Sans KR | 한글 가독성 최적 |
| 디자인 | CSS Variables 기반 다크 테마 | 커스터마이징 용이 |
| 상태 | Vanilla JS Module Pattern | 의존성 0 |

> **향후 백엔드 전환 권장 스택**: FastAPI (Python) + PostgreSQL 16 + Redis + Keycloak SSO

## 🔒 보안 및 개인정보

- **RBAC**: 역할별 대시보드 접근 제한 (7단계 역할)
- **데이터 익명화**: 통계·연구용 화면은 k-익명성 적용
- **감사 추적**: 모든 데이터 변경 이력 audit_logs 테이블 자동 기록
- **PII 보호**: 현재 버전은 Mock 데이터 사용 — 실제 배포 시 AES-256 암호화 필수

> ⚠️ 실제 운영 배포 전 **개인정보 영향 평가(PIA)** 실시 필수 (개인정보보호법 제33조)
